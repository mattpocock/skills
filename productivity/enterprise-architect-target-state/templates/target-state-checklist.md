# Target-State Architecture Checklist

Use this checklist before finalizing a target-state architecture.

## Business alignment
- [ ] Business outcome is explicit
- [ ] Scope is clear
- [ ] Target state supports growth assumptions
- [ ] Recommendation is tied to business value

## Current state
- [ ] Current-state constraints identified
- [ ] Major pain points documented
- [ ] Relevant systems and domains included
- [ ] Dependencies understood

## Target state
- [ ] Target-state design is specific
- [ ] Principles are stated clearly
- [ ] Preferred patterns are named
- [ ] Anti-patterns are called out

## Transition
- [ ] Migration phases are defined
- [ ] Critical dependencies are sequenced
- [ ] Legacy retirement path is included
- [ ] Interim states are realistic

## Governance
- [ ] Applicable standards identified
- [ ] Security / compliance implications covered
- [ ] Approval path noted
- [ ] Exceptions tracked if needed

## Decision quality
- [ ] Tradeoffs are explicit
- [ ] Assumptions are stated
- [ ] Risks are surfaced
- [ ] Recommendation is actionable

## Executive readiness
- [ ] Summary is concise
- [ ] Language is understandable to non-architects
- [ ] Next actions are clear
- [ ] Output can be used in a review or steering meeting
