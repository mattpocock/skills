# Launch Checklist Template

## Before launch
- [ ] Problem statement and success metrics are clear
- [ ] Implementation is complete and reviewed
- [ ] Tests pass
- [ ] Monitoring / observability exists
- [ ] Rollback plan exists
- [ ] Support plan exists
- [ ] Comms plan exists

## Launch day
- [ ] Release is enabled for the intended segment
- [ ] Metrics are being watched
- [ ] Error rates are stable
- [ ] Feedback channel is monitored
- [ ] Rollback criteria are defined

## After launch
- [ ] Check metric movement
- [ ] Check user feedback
- [ ] Check support tickets / errors
- [ ] Decide whether to expand, iterate, or roll back

## Post-launch review
- What worked?
- What failed?
- What should change next?
